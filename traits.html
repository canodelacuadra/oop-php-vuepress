# ¿ Qué son los taits o rasgos?

PHP solo admite herencia única: una clase secundaria puede heredar solo de un padre único.

Entonces, ¿qué pasa si una clase necesita heredar múltiples comportamientos? Los rasgos de OOP resuelven este problema.

Los rasgos se utilizan para declarar métodos que se pueden utilizar en varias clases. Los rasgos pueden tener métodos y métodos abstractos que se pueden usar en múltiples clases, y los métodos pueden tener cualquier modificador de acceso (público, privado o protegido).

Los rasgos se declaran con la palabra clave trait :

Sintaxis

<?php
trait TraitName {
  // some code...
}
?>

Para usar un rasgo en una clase, usamos la palabra clave use:

Sintaxis

<?php
class MyClass {
  use TraitName;
}
?>

Veamos un ejemplo:

Ejemplo

<?php
trait mensajes {
public function mensaje1() {
    echo "OOP es divertida! ";
  }
}

class Bienvenida {
  use mensaje;
}

$obj = new Bienvenida();
$obj->mensaje1();
?>

Hemos declarado un rasgo: mensaje1. Luego, creamos una clase: Bienvenida. La clase usa el rasgo y todos los métodos del rasgo estarán disponibles en la clase.

Si otras clases necesitan usar la función mensaje1(), simplemente use el rasgo message1 en esas clases. Esto reduce la duplicación de código, porque no es necesario volver a declarar el mismo método una y otra vez.

# PHP: uso de múltiples traits

Veamos otro ejemplo:

<?php
trait message1 {
  public function msg1() {
    echo "OOP is fun! ";
  }
}

trait message2 {
  public function msg2() {
    echo "OOP reduces code duplication!";
  }
}

class Welcome {
  use message1;
}

class Welcome2 {
  use message1, message2;
}

$obj = new Welcome();
$obj->msg1();
echo "<br>";

$obj2 = new Welcome2();
$obj2->msg1();
$obj2->msg2();
?>

Aquí, declaramos dos rasgos: mensaje1 y mensaje2. Luego, creamos dos clases: Bienvenida y Bienvenida2. La primera clase (Bienvenido) usa el rasgo message1, y la segunda clase (Welcome2) usa los rasgos message1 y message2 (los rasgos múltiples están separados por comas).
